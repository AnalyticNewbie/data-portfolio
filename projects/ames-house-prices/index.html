Ames Housing Price Prediction

Kaggle “House Prices: Advanced Regression Techniques” — a validation-first approach focused on robust preprocessing, residual analysis, and Optuna-tuned CatBoost regularisation.

Overview

This project solves the Kaggle House Prices: Advanced Regression Techniques task using a disciplined validation-first workflow. Instead of relying on feature churn or leaderboard-specific tricks, the focus was on building a generalisable pipeline and improving performance through targeted model regularisation.

Problem framing

The dataset contains ~1,460 training rows with mixed numerical and categorical features, a skewed target distribution, and a small number of extreme outliers. The RMSLE metric makes tail behaviour and generalisation more important than raw in-sample fit.

Approach

  • Preprocessing: domain-aware imputation, log1p target transform, skewness correction, ordinal encoding, and conservative outlier handling.
  • Validation discipline: consistent 5-fold CV with frozen splits; decisions based on out-of-fold performance.
  • Residual-driven analysis: used to diagnose failure modes and avoid adding high-variance features that reduce generalisation.
  • Hyperparameter optimisation: Optuna tuning of CatBoost regularisation and stochasticity (Bayesian bootstrap + early stopping).
Key outcome: the biggest improvement came from tuning regularisation, not adding more features. Feature-heavy variants (e.g., neighbourhood target encodings) improved CV but degraded public LB — a clear generalisation warning.

Final model

A single CatBoost model with Optuna-tuned hyperparameters. Tuning used frozen folds and early stopping to minimise overfitting and stabilise tail predictions.

Best tuned parameters (high-level): shallow trees (depth=4), moderate L2 regularisation, Bayesian bagging with tuned bagging temperature, and controlled split randomness.

Key learnings

  • Regularisation beats feature churn on small, mixed-type tabular datasets.
  • Residual analysis is diagnostic — not every pattern should become a feature.
  • CV ↔ LB divergence is a trap; frozen folds and conservative changes prevent chasing noise.
  • Simpler models can generalise better than stacked ensembles if they are well-tuned.
