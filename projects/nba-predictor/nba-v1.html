v1.0: MVP Launch

The Proof of Concept. Before the databases and the complex pipelines, there was just a Python script and a folder full of CSVs.

The Goal

To answer a simple question: Can I automate the collection of NBA data and identify basic value discrepancies against Vegas lines using a simple script?

Hypothesis

"A basic automated system should outperform random guessing."

The Approach

Python 3.12 Pandas CSV Storage Linear Regression

We started with zero infrastructure. No database, no cloud hosting. Just a local script pulling data via nba_api and storing it in flat CSV files.

Key Lessons & Limitations

No Memory

Without a database, the script had to recalculate every feature from scratch every time it ran. It was inefficient and slow.

Too Simple

Relying solely on "Points Scored" failed to account for context. 100 points against a fast team is very different than 100 points against a defensive lock.

Data Fragility

CSVs are prone to corruption. Querying complex matchups (e.g., "How do the Lakers do on Mondays?") was nearly impossible.

The Evolution

The MVP proved the concept, but the infrastructure couldn't scale. To move forward, we needed a real database and a model that understands non-linear factors like fatigue.
