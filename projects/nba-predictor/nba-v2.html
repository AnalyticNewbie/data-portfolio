v2.0: The Great Migration

From CSVs to SQL and Linear Regression to Gradient Boosting.

1. SQL Feature Engineering

Calculating "Rolling 10-Game Averages" in Python was becoming slow. I migrated the data to PostgreSQL and used Window Functions to calculate features instantly. 

-- The View that powers the model
CREATE VIEW v_team_features AS
SELECT 
    team_id,
    AVG(pts) OVER (
        PARTITION BY team_id 
        ORDER BY game_date 
        ROWS BETWEEN 4 PRECEDING AND CURRENT ROW
    ) as rolling_points_5
FROM team_stats;

2. The Brain Transplant

Linear Regression assumes relationships are straight lines. Basketball is non-linear (e.g., Fatigue hurts good teams more than bad teams). I switched to Gradient Boosting.
