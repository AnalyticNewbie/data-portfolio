The Shift

v1.0 was a good proof of concept, but it lacked nuance. It couldn't account for complex interactions like "Strength of Schedule" or "Rest Days."

In v2.0, I moved the entire feature engineering pipeline into PostgreSQL and implemented scikit-learn pipelines for the modeling.

Key Architectures

1. SQL-First Engineering

Instead of calculating features in Python loops (which is slow and error-prone), I created SQL Views. This allows me to calculate complex rolling averages and "Strength of Schedule" adjustments instantly at the database level.

2. The Truth Loop

I built a feedback loop where every prediction is logged to a history table (`prediction_history`). A separate script runs daily to ingest actual scores, grade the predictions, and track model drift over time.

3. Gradient Boosting

I utilized HistGradientBoostingRegressor for the score predictions. This model handles non-linear relationships better than linear regression, allowing it to capture how "Rest Disadvantage" impacts scoring differently for elite vs. average teams.
