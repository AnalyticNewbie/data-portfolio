v2.1: The Sunday Fix

Post-migration stabilization. Addressing "Silent Failures" in the pipeline and the critical Timezone ingestion error that skewed weekend predictions.

The Objective

Stabilization

Real-world usage of v2.0 exposed a bottleneck: Saturday night games were bleeding into Sunday morning data frames due to UTC offsets, causing the model to miscalculate "Rest Days."

  • Implement UTC to EST normalization
  • Create "Safe State" fallbacks for API timeouts

The Strategy

Surgical Refactoring

Unlike the v2.0 overhaul, this update was targeted. We used a "Strangler" approach to refactor only the failing date-utility modules without risking the entire ingestion engine.

Success Metric
0% Missing Games

The "Sunday Bug"

The Problem

The API returned UTC timestamps. A 10 PM game in LA on Saturday was recorded as Sunday 1 AM. This caused our "Rest Disadvantage" feature to falsely think teams had zero days off.

// Hacky v2.0 code (Bad)
df['game_date'] = pd.to_datetime(df['GAME_DATE'])
# Fails on West Coast games...

The Solution

Implemented localized timezone aware objects. We now "snap" all games to the arena's local timezone before calculating the rest-delta.

// Refined v2.1 code (Surgical)
def localize_date(row):
  utc_dt = to_utc(row['date'])
  return utc_dt.astimezone('US/Eastern')

Launch Retrospective

User Confusion

Moving to a dashboard made users miss the raw data tables. v2.1 added "View Source" buttons for every prediction to build trust.

Silent Failures

The API integration failed silently on timeouts. We implemented "Loading Skeletons" and error boundaries to prevent a white-screen crash.

Mobile Breakage

The new AUC charts didn't scale well on small viewports (iPhone SE). Refactored charts to be responsive and touch-friendly.
