NBA Game Prediction System

An end-to-end NBA forecasting pipeline that predicts win probability, home/away scores, and simulation-based 95% prediction intervals, refreshed daily with ET-correct ingestion.

Python 3.13 Postgres 16 Docker nba_api SQL Feature Store Uncertainty / Intervals

What it does

  • Ingests NBA schedule and final results daily, aligned to NBA Eastern Time (ET).
  • Stores game/team data in PostgreSQL (Docker) and materialises engineered features for each scheduled game.
  • Trains baseline models for:
    • Win probability (logistic regression)
    • Score margin and total points (gradient boosting regressors)
  • Produces home/away score means and 95% prediction intervals using simulation.

Architecture

NBA API
  ├─ Schedule (future games)
  └─ LeagueGameLog (completed games)
        ↓
Python ingestion (ET-correct)
        ↓
Postgres (Docker) + SQL feature engineering
        ↓
Training views (3-year time-decay weighting)
        ↓
Python models (win%, margin, total)
        ↓
Daily predictions + simulation-based 95% intervals

Time zones are handled explicitly: ET is canonical in the database (game_date_et), while user input dates (e.g., AEDT) are converted to ET for querying.

Feature engineering (baseline)

Recent form

Rolling point differential over 5 and 10 games (materialised onto scheduled games).

Fatigue / rest context

Rest days per team based on the most recent final game prior to the scheduled game. Back-to-backs are detected as 0 rest days.

Home advantage

Simple home indicator (baseline), designed to be extended with richer context.

Time-decay weighting

Training data covers the last 3 seasons with recency weighting to reflect modern play style.

Uncertainty (95% prediction intervals)

Score intervals are simulation-based rather than simple error bounds. The system estimates residual uncertainty from a time-split validation set and simulates thousands of outcomes per game to compute the 2.5th and 97.5th percentiles for home and away scores. 

margin ~ Normal(μ_margin, σ_margin)
total  ~ Normal(μ_total,  σ_total)

home = (total + margin)/2
away = (total - margin)/2

95% PI = percentile(home, [2.5, 97.5]) and percentile(away, [2.5, 97.5])
